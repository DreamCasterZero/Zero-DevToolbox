# 项目约定（Python / 算法）

## 沟通

- 用中文回复。
- 解释代码时先讲思路，再看实现。
- 涉及技术取舍时，给出多个方案及各自的优劣，不要默默替我决定。

## 环境与依赖

- 用 conda 管理环境，不要用 pip+venv 或 uv 的指令。
- 涉及安装依赖时，给 conda 命令；需要 pip 安装的包，在 conda 环境内用 pip。

## 代码风格

- 核心函数（算法逻辑、对外接口）要写类型注解和清晰的输入输出说明；临时脚本、内部小函数可不强制。
- 算法核心逻辑与可视化、文件读写、参数配置分离，不要混在一个函数里。
- 性能敏感的数值计算优先用 numpy 向量化，避免不必要的 Python 显式循环。

## 算法代码

- 涉及随机性的算法（GA、SA、ALNS、PSO 等）必须支持固定随机种子，保证结果可复现。
- 算法实现应能用一个小规模算例手算验证正确性；必要时附上这样的最小示例。
- 关键算法函数的注释要写清楚核心思路和方法依据，而不只是描述代码在做什么。
